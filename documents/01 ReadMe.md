## üöÄ Getting Started

### Step 1: Today (Team Formation)

1. Form a team of 5 people
2. Fill out **TEAM_REGISTRATION_FORM.md** together
3. Decide who codes which component
4. Submit form to instructor by end of class
5. Download all project files

### Step 2: Week 1 (Build Your Component)

1. Read **ASSIGNMENT.md** completely
2. Find your assigned component file (e.g., student.py)
3. Complete all TODO sections
4. Test your code using the test code at the bottom
5. Fill in your name/ID in the file header

### Step 3: GitHub Setup

1. Create private GitHub repository
2. Add instructor as collaborator
3. Upload your team's code
4. See **GITHUB_REQUIREMENTS.md** for details

### Step 4: Week 2 (Peer Review)

1. Wait for instructor to announce reviewer assignments (Dec 5)
2. Review your assigned component thoroughly
3. Use **PEER_REVIEW_FORM.md** template
4. Test the code, document issues
5. Author fixes issues based on your feedback

### Step 5: Final Submission

1. Integrate all fixed components
2. Test complete system
3. Complete **README_TEMPLATE.txt**
4. Include all peer review forms
5. Submit GitHub repository link to Brightspace

## ‚ùì Common Questions

**Q: Do I get graded on GitHub usage?**
A: No. GitHub is just how you submit. You're graded on code, reviews, and integration.

**Q: Why is review worth 50%?**
A: In real software development, QA/code review is more valuable than writing code.

**Q: Can we choose our reviewer?**
A: No. Reviewers are randomly assigned on Dec 5 to ensure fairness.

**Q: What if someone doesn't do their work?**
A: Tell instructor immediately. Team won't be penalized.

**Q: Is LinkedIn Learning required?**
A: No, it's optional but helpful if you're new to Git/GitHub.
