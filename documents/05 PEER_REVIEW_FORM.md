## Review Checklist

### Code Functionality (Check all that apply)

- [ ] Code runs without errors
- [ ] All required methods are implemented
- [ ] Methods work as specified in documentation
- [ ] Test code at bottom of file runs successfully
- [ ] Edge cases are handled (empty input, invalid data, etc.)
- [ ] File I/O works correctly (if applicable)
- [ ] Integration with other components works

### Code Quality

- [ ] Code follows PEP 8 style guidelines
- [ ] Variable names are meaningful and descriptive
- [ ] Proper indentation (4 spaces)
- [ ] No unnecessary code duplication
- [ ] Comments explain complex logic
- [ ] Code is organized and readable

### Documentation

- [ ] Author name, student ID, and date filled in at top
- [ ] All methods have docstrings
- [ ] Docstrings include author and date
- [ ] Docstrings explain parameters and return values
- [ ] Purpose of class is clearly documented

### Validation & Error Handling

- [ ] User input is validated appropriately
- [ ] File errors are handled (FileNotFoundError)
- [ ] Helpful error messages displayed
- [ ] Program doesn't crash on invalid input
